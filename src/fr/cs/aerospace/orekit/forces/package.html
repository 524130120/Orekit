This package provides interfaces that will be used by the {@link fr.cs.aerospace.orekit.propagation.NumericalPropagator}.

The {@link fr.cs.aerospace.orekit.forces.ForceModel} instance is added the the propagator before the extrapolation. The propagator calls at each step the addContribution method of the force model, giving all the state information it needs : date, position, velocity, frame, attitude, mass and above all the TimeDerivativeEquations instance. The selected force model calculates then the acceleration relative to the model and state, and adds it to the {@link fr.cs.aerospace.orekit.propagation.TimeDerivativesEquations}.

A useful object has been developed to handle the discontinuities of the acceleration (typically for maneuvers or solar drag for instance): the {@link fr.cs.aerospace.orekit.forces.SWF switching functions}. The g function method is called at each step and must return a double. The integrator is looking for its change of sign, it means that a particular step must coincide with the g function sign change, so that no important event can be missed. A force model can use several switching functions or not at all.
