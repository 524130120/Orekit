This package provides classes to represent orbits and orbital parameters. It builds the base of all the other space mechanics tools. It proposes an independent interface, {@link fr.cs.aerospace.orekit.orbits.OrbitalParameters}, implemented in four different ways, and {@link fr.cs.aerospace.orekit.orbits.Orbit orbits}, which are orbital parameters associated to a date.

The user must be aware of the correct definition of these parameters and of the singularities relative to each one of them (for Keplerian and circular elements, particularly).

The way it is handled in OREKIT is very simple and permits easy and transparent conversions. All 4 parameters type realize the abstract class {@link fr.cs.aerospace.orekit.orbits.OrbitalParameters}, and each extrapolation tool uses this interface as parameter. Some tools can convert them for internal calculation ( the NumericalPropagator, for instance, uses internally the equinoctial parameters to avoid singularities ). But each type of orbital parameters can return every non-ambiguous parameter (a, e, i, position and velocity, etc.) or can be converted so the user has not to always worry about the kind of parameters he manipulates.
